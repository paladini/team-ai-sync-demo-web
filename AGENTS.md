# Shared AI Agent Instructions

These instructions are synchronized from `paladini/team-ai-sync-demo-source`.

## Working Agreement

- Read the repository README before changing code.
- Prefer small, reviewable changes with focused commits.
- Use clear commit messages that describe the intent of the change.
- Keep generated explanations concise and tied to the files changed.
- When behavior changes, update tests or explain why tests are not applicable.
- Do not commit secrets, credentials, personal tokens, or machine-local paths.

## Review Checklist

- Confirm the change solves the stated problem.
- Check whether documentation or examples need to move with the code.
- Verify that formatting and tests run with the repository's documented commands.
